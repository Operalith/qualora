# Qualora v0.1.0-alpha

Qualora v0.1.0-alpha is the first public alpha of Qualora: open-source, self-hosted autonomous QA for web applications and APIs.

This is an early MVP. It is useful for local browser smoke checks and structured reports, but it is not a complete QA platform yet.

## What Works

- Run the stack locally with Docker Compose.
- Create a QA project through the API.
- Start a browser smoke run.
- Execute the run with Playwright Chromium.
- Enforce `allowed_hosts` for project targets and browser requests.
- Capture page title, screenshot evidence, console errors, failed requests, and blocked out-of-scope requests.
- Store metadata in PostgreSQL.
- Queue runs with Redis.
- Store screenshots in MinIO/S3.
- Fetch structured JSON reports.
- Run an end-to-end smoke script against `https://example.com`.

## Quick Start

```bash
docker compose up -d --build
make smoke
```

If local port `8080` is already in use:

```bash
QUALORA_API_PORT=18080 docker compose up -d --build
QUALORA_API_URL=http://localhost:18080 make smoke
```

## Safety Notes

Only run Qualora against systems you own or are explicitly authorized to test.

The alpha blocks localhost, loopback, link-local, private IP literal targets, common cloud metadata targets, and public hostnames resolving to blocked IP ranges by default. Use `allow_private_targets: true` only for private systems you control.

## Not Included Yet

- Web UI.
- API worker.
- OpenAPI contract testing.
- Authentication.
- Login automation.
- Active security scanning.
- Helm/Kubernetes deployment.

## Recommended Next Steps
