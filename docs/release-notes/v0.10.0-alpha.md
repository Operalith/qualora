# Qualora v0.10.0-alpha

`v0.10.0-alpha` adds the first role-aware authorization testing foundation.

## Highlights

- Credential profiles now support optional role metadata.
- Users can create explicit browser URL authorization checks.
- Authorization runs log in with the configured actor profile and navigate only to the configured same-origin target.
- Reports show expected vs actual authorization outcomes, findings, screenshots, and evidence metadata.
- The web UI includes authorization check creation, run history, and report pages.
- The deterministic demo web target includes admin, readonly, customer-a, and customer-b accounts.

## Safety Scope

Authorization checks are conservative and read-only:

- No crawling.
- No fuzzing.
- No payload-based exploitation.
- No destructive actions.
- No arbitrary form submission.
- No autonomous AI browser control.
- No credential, cookie, storage, auth header, or token exposure in reports or AI inputs.

## Known Limitations

- No Qualora user authentication or authorization.
- Browser URL authorization checks are supported first.
- Authenticated API authorization testing is not fully supported yet.
- The UI remains alpha and should be used only in trusted local/self-hosted environments.
