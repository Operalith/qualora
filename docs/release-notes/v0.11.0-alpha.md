# Qualora v0.11.0-alpha

`v0.11.0-alpha` adds local authentication for self-hosted Qualora instances.

## Highlights

- First-run setup creates one local admin account.
- The web UI now shows setup, login, authenticated app, and logout states.
- Project data, credential profiles, AI provider config, reports, evidence, API specs, test plans, and authorization checks are protected by default.
- Passwords are hashed with Argon2id.
- Sessions use an HTTP-only cookie and CSRF protection for mutating API calls.
- Smoke tests now set up and log in as `admin@qualora.local` before running the existing end-to-end demo flow.

## Security Notes

- Health, setup status, setup admin, login, logout, and auth/me endpoints are public.
- Evidence downloads and JSON/HTML reports require authentication.
- Session tokens are stored only as hashes in PostgreSQL.
- Do not expose the alpha UI/API to untrusted networks without a trusted deployment boundary.

## Known Limitations

- One local admin role only.
- No multi-tenancy.
- No SSO/OIDC/SAML.
- No advanced RBAC or user management UI.
- No built-in login rate limiting or audit log yet.
