# Qualora v0.12.0-alpha

`v0.12.0-alpha` adds safe deterministic application discovery and a persistent application map.

## Highlights

- Start discovery from a project with `frontend_url`.
- Store discovery runs, pages, links, forms, fields, findings, and evidence metadata.
- Follow safe same-origin links by default with `max_pages=20` and `max_depth=2`.
- Record skipped unsafe, external, unsupported-scheme, and non-HTML links.
- Capture screenshots and browser observations for discovered pages.
- View discovery runs and application maps in the web UI.
- Export discovery JSON and self-contained HTML reports.
- Smoke test discovery against the deterministic local `demo-web` service.

## Safety

Discovery is deterministic and conservative. It does not submit forms, click arbitrary buttons, execute payloads, perform destructive actions, crawl external domains by default, or use autonomous AI browser control.

Discovery records metadata and screenshots only. It must not store full HTML, cookies, local/session storage, auth headers, tokens, credentials, request bodies, or response bodies.

## Known Limitations

- Discovery is bounded alpha coverage, not exhaustive crawling.
- Routes that require arbitrary button clicks or form submission are not explored.
- Discovery reports are not automatically used for AI analysis in this release.
- Authenticated discovery depends on an explicitly selected project credential profile and the existing deterministic selector-based login flow.
