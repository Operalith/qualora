# Qualora v0.13.0-alpha

`v0.13.0-alpha` adds discovery-aware AI test generation and Safe QA Runs.

## Highlights

- Generate AI-assisted test plans from sanitized application discovery maps.
- Ask AI for safe executable browser DSL candidates while keeping plans reviewable.
- Persist test plan source metadata, linked discovery run IDs, and safe execution coverage.
- Start Safe QA Runs that reuse an existing discovery run, use the latest completed discovery run, or create a new bounded discovery run.
- Preview safe execution by default before any browser actions run.
- Explicitly execute only the persisted, mapped, non-destructive browser DSL path.
- View Safe QA Run JSON and self-contained HTML reports.
- Use the web UI to generate discovery-aware plans, start Safe QA Run previews, execute reviewed runs, and inspect reports.
- Smoke test discovery-aware planning and Safe QA Run preview/execution against local `demo-web` and `fake-llm`.

## Safety

Safe QA Runs are orchestration, not autonomous AI browser control. AI can suggest reviewable plans from sanitized metadata, but Qualora executes only supported deterministic DSL steps after preview and explicit user action.

Discovery-aware AI input excludes credentials, cookies, local/session storage, auth headers, tokens, full HTML, screenshots, request bodies, and response bodies.

Safe QA Runs do not click arbitrary buttons, submit forms, run payloads, fuzz inputs, perform active scans, crawl beyond the configured discovery policy, or execute destructive actions.

## Known Limitations

- AI planning is optional and requires a configured OpenAI-compatible provider.
- Safe QA Run execution remains browser-only and limited to the supported safe DSL.
- The workflow depends on discovery coverage and sanitized metadata quality.
- Authenticated discovery and multi-step logged-in journeys are not included.
- No active security scanning, fuzzing, arbitrary form submission, or autonomous AI browser control.
