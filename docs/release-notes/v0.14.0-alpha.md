# Qualora v0.14.0-alpha

`v0.14.0-alpha` adds passive front-end quality checks for safe security, accessibility, and performance signals.

## Highlights

- Start standalone quality check runs for a project frontend.
- Reuse the latest or selected completed discovery run as the quality-check page list.
- Optionally authenticate with a project-scoped credential profile before checks.
- View quality summaries and findings in the web UI.
- Export JSON and self-contained HTML quality reports.
- Optionally include passive quality checks in Safe QA Runs.

## Safety Scope

Quality checks are passive and deterministic. They do not submit forms, click arbitrary buttons, run payloads, fuzz inputs, perform active security scanning, perform destructive actions, crawl external domains broadly, or use autonomous AI browser control.

Quality evidence stores metadata only. It must not include cookie values, browser storage, authorization headers, tokens, secrets, request bodies, response bodies, full HTML, or credentials.

## Known Limitations

- This is alpha heuristic coverage, not a full security assessment.
- Accessibility checks are basic metadata heuristics and are not WCAG certification.
- Performance checks use browser/resource observations only; Lighthouse and Core Web Vitals are not implemented yet.
- Authenticated quality checks support only deterministic selector-based login through credential profiles.
