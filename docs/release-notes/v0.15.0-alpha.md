# Qualora v0.15.0-alpha

`v0.15.0-alpha` adds guided real-world onboarding for first project setup. The release focuses on making existing safe capabilities easier to discover and run from a self-hosted local UI.

## Highlights

- New guided project setup wizard at `#/setup-project`.
- New onboarding API endpoint: `POST /api/v1/onboarding/project-setup`.
- Improved dashboard with version badge, quick-start actions, status indicators, recent projects, and recent Safe QA runs.
- Local demo workflow that can create a demo project, configure the fake LLM provider, import the demo OpenAPI spec, create a demo credential profile, and start selected safe checks.
- Project readiness checklist on project detail pages.
- Reports landing page at `#/reports`.
- Smoke coverage for guided setup, browser smoke, authenticated smoke, discovery, quality checks, Safe QA, API smoke, report links, UI discoverability, and secret redaction.

## Safety

- Guided setup orchestrates existing safe features only.
- Destructive actions remain disabled.
- AI remains optional.
- Credentials are encrypted at rest and never sent to AI.
- Guided setup responses do not return raw passwords, provider secrets, cookies, browser storage, authorization headers, tokens, or encrypted secret payloads.
- Safe QA remains previewable and deterministic; generated plans are not executed as free-form AI browser control.

## Known Limitations

- The wizard is alpha and intentionally minimal.
- Demo workflow is intended for the local Docker Compose demo services.
- Reports index focuses on recent browser/API smoke, discovery, quality, and Safe QA reports.
- No autonomous AI browser control.
- No arbitrary form submission.
- No destructive testing.
- No active security scanning.
- No fuzzing.
- No SSO/OIDC/SAML, multi-tenancy, or enterprise RBAC.
- Only OpenAI-compatible chat completion providers are supported.
