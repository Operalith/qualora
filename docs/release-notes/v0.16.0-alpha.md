# Qualora v0.16.0-alpha

`v0.16.0-alpha` adds Interactive Safe Explorer, a deterministic browser workflow for bounded, human-readable exploration of safe navigation actions.

## Highlights

- New Safe Explorer API:
  - `POST /api/v1/projects/{project_id}/safe-explorer-runs`
  - `GET /api/v1/projects/{project_id}/safe-explorer-runs`
  - `GET /api/v1/safe-explorer-runs/{safe_explorer_run_id}`
  - `GET /api/v1/safe-explorer-runs/{safe_explorer_run_id}/trace`
  - `GET /api/v1/safe-explorer-runs/{safe_explorer_run_id}/report`
  - `GET /api/v1/safe-explorer-runs/{safe_explorer_run_id}/report.html`
- Browser worker support for bounded Safe Explorer jobs.
- Persistent Safe Explorer run, step, and action records.
- Visible action extraction for links, forms, submit controls, buttons, and inputs without storing full HTML.
- Deterministic action classification and skip reasons.
- Safe navigation execution only by default.
- Web UI project form, run list, report page, and reports index integration.
- Demo and smoke coverage for safe executed actions, skipped unsafe/external/POST/unsupported actions, screenshots, reports, and redaction.

## Safety Scope

Safe Explorer does not use AI to choose or execute actions. It does not submit POST forms, fill arbitrary forms, click arbitrary buttons, run payloads, fuzz inputs, perform active security scanning, or perform destructive actions.

Safe Explorer skips destructive-looking links, external targets, unsupported schemes, sensitive query URLs, mutating forms, password/file/hidden-sensitive forms, unsupported controls, duplicates, and policy-limited actions with explicit reasons.

## Known Limitations

- Alpha coverage only; this is not exhaustive crawling.
- GET forms are disabled by default.
- POST/PUT/PATCH/DELETE-style form execution is not supported.
- Safe QA Run integration is not included in this release.
- No autonomous AI browser control was added.
