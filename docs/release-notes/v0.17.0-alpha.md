# Qualora v0.17.0-alpha

`v0.17.0-alpha` adds deterministic report intelligence so teams can understand alpha QA output faster without hiding raw details.

## Highlights

- Executive summaries for JSON and HTML reports.
- Severity normalization to `critical`, `high`, `medium`, `low`, and `info`.
- Deterministic finding fingerprints and grouped findings.
- Duplicate reduction and repeated/noisy finding summaries.
- Top findings and affected-page summaries.
- Report detail UI panels for executive summary, top findings, grouped findings, affected pages, and noise/repeated signals.
- Reports landing page now shows recent status, high/medium counts, grouped counts, and raw counts.
- Smoke coverage for report intelligence fields, HTML sections, grouped findings, raw findings/results, and secret redaction.

## Report Fields

Primary report responses now include:

- `executive_summary`
- `severity_counts`
- `grouped_findings`
- `top_findings`
- `top_affected_pages`
- `noise_summary`
- `raw_findings_count`
- `deduplication_summary`
- `safety_limitations`

Raw findings, quality result rows, evidence metadata, discovery maps, Safe Explorer traces, authorization results, API results, and safe test plan execution details remain available.

## Safety

Report intelligence is deterministic and does not call an AI provider. It uses already persisted findings, safe result rows, and metadata only.

It must not include credentials, cookies, local/session storage, authorization headers, tokens, screenshots, full HTML, request bodies, response bodies, provider secrets, or encrypted secret payloads. Sensitive URL query parameters are redacted before grouping or fingerprinting.

## Known Limitations

- Finding grouping is heuristic and deterministic, not a root-cause engine.
- AI report briefs are not included in this release.
- No active security scanning, schema fuzzing, destructive testing, autonomous AI browser control, arbitrary form submission, multi-tenancy, SSO/OIDC/SAML, or production hardening was added.
- Reports remain alpha guidance and are not exhaustive QA certification.
