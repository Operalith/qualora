# Qualora v0.18.0-alpha

`v0.18.0-alpha` adds deterministic Safe QA report baselines, regression comparison, and CI-friendly quality gates.

## Highlights

- Mark a completed Safe QA report as a project baseline.
- Keep one default baseline per project and report type.
- Compare a later Safe QA report against the default or selected baseline.
- Classify grouped findings as new, fixed, unchanged, severity-changed, or affected-scope-changed.
- Evaluate alpha quality gates for new and total severity thresholds.
- Return compact CI JSON with an exit code through `format=ci`.
- Use `scripts/qualora-ci-gate.sh` for HTTP-based CI checks.
- View baseline, comparison, and quality gate status in Safe QA JSON/HTML reports and the web UI.
- Smoke coverage for baseline creation, unchanged comparison, passing gate, compact CI output, HTML report text, UI text, and no-secret report output.

## API Endpoints

- `POST /api/v1/projects/{project_id}/report-baselines`
- `GET /api/v1/projects/{project_id}/report-baselines`
- `GET /api/v1/report-baselines/{baseline_id}`
- `PUT /api/v1/report-baselines/{baseline_id}`
- `DELETE /api/v1/report-baselines/{baseline_id}`
- `POST /api/v1/projects/{project_id}/report-comparisons`
- `POST /api/v1/projects/{project_id}/quality-gates/evaluate`

## CI Example

```bash
QUALORA_API_URL=http://localhost:8080 \
QUALORA_PROJECT_ID="$PROJECT_ID" \
QUALORA_REPORT_ID="$QA_RUN_ID" \
QUALORA_SESSION_COOKIE="qualora_session=..." \
QUALORA_CSRF_TOKEN="$CSRF" \
scripts/qualora-ci-gate.sh
```

The script exits `0` for passed or warning gates and `1` for failed gates.

## Safety

Baselines and quality gates are deterministic control-plane features. They do not require AI, start workers, execute browser/API checks, perform active scans, fuzz inputs, run payloads, or mutate tested systems.

Baseline and comparison data must not include credentials, cookies, local/session storage, authorization headers, tokens, screenshots, full HTML, request bodies, response bodies, provider secrets, encrypted secret payloads, or raw AI prompts.

## Known Limitations

- Baseline comparison is alpha and fingerprint-based.
- Safe QA is the primary supported baseline report type in this release.
- Quality gates are conservative release signals and do not replace human review.
- CI integration is HTTP/script based, not a full CLI.
- No autonomous AI browser control, active security scanning, payload attacks, fuzzing, destructive testing, hosted SaaS assumptions, multi-tenancy, SSO/OIDC/SAML, or production hardening was added.
- Only OpenAI-compatible chat completion providers are supported for optional AI features.
