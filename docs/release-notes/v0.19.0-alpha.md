# Qualora v0.19.0-alpha

`v0.19.0-alpha` adds native CI mode, pipeline-friendly run orchestration, and optional sanitized GitHub/GitLab issue export.

## Highlights

- Start a CI run with `POST /api/v1/projects/{project_id}/ci-runs`.
- Reuse the latest completed Safe QA report with `run_safe_qa=false` for AI-free quality gate evaluation.
- Start the existing Safe QA workflow with `run_safe_qa=true`, compare with a baseline, evaluate gates, and return an exit code.
- Persist CI run summaries in PostgreSQL for project/report visibility.
- Use `scripts/qualora-ci-run.sh` for run-and-gate CI workflows.
- Continue using `scripts/qualora-ci-gate.sh` for existing-report quality gates.
- Configure optional GitHub/GitLab issue export targets with encrypted tokens.
- Dry-run issue export previews from grouped sanitized findings before creating tracker issues.
- Manage CI runs and issue export from the alpha web UI.
- Smoke coverage for native CI, both helper scripts, issue export dry-runs, UI text, and secret redaction.

## CI Examples

Evaluate an existing Safe QA report:

```bash
QUALORA_API_URL=http://localhost:8080 \
QUALORA_EMAIL=admin@qualora.local \
QUALORA_PASSWORD="$QUALORA_PASSWORD" \
QUALORA_PROJECT_ID="$PROJECT_ID" \
QUALORA_REPORT_ID="$QA_RUN_ID" \
QUALORA_BASELINE_ID="$BASELINE_ID" \
scripts/qualora-ci-gate.sh
```

Run or reuse a CI workflow:

```bash
QUALORA_URL=http://localhost:8080 \
QUALORA_EMAIL=admin@qualora.local \
QUALORA_PASSWORD="$QUALORA_PASSWORD" \
QUALORA_PROJECT_ID="$PROJECT_ID" \
QUALORA_BASELINE_ID="$BASELINE_ID" \
QUALORA_RUN_SAFE_QA=false \
scripts/qualora-ci-run.sh
```

Both scripts print compact JSON and exit `0` for passed/warning results or `1` for failed/error results.

## Issue Export

Issue export is optional and dry-run by default. Configured tokens are encrypted with `QUALORA_ENCRYPTION_KEY` and are never returned raw. Export content is generated from grouped sanitized finding metadata only: title, severity, category, affected page count, representative paths, recommendation, report link, safety note, fingerprint, labels, and Qualora version.

Real issue creation requires an enabled config, a configured token, and `dry_run=false`.

## Safety

Native CI runs, quality gates, and issue export do not add a new testing engine. They orchestrate existing Safe QA reports, deterministic baselines, deterministic comparison, and deterministic quality gates.

CI output and issue export content must not include local admin passwords, session cookies, CSRF tokens, tracker tokens, provider secrets, credential profile secrets, cookies, browser storage, authorization headers, screenshots, full HTML, request bodies, response bodies, raw logs, encrypted secret payloads, or raw AI prompts.

## Known Limitations

- Native CI mode is alpha and centers on Safe QA reports.
- New Safe QA runs still require an OpenAI-compatible provider when `run_safe_qa=true`.
- AI-free CI evaluation requires an existing completed Safe QA report.
- Issue export supports GitHub/GitLab-compatible issue creation only.
- Issue export does not sync, reopen, close, or deduplicate against existing tracker issues.
- No full CLI, autonomous AI browser control, active security scanning, payload attacks, fuzzing, destructive testing, hosted SaaS assumptions, multi-tenancy, SSO/OIDC/SAML, or production hardening was added.
- Only OpenAI-compatible chat completion providers are supported for optional AI features.
