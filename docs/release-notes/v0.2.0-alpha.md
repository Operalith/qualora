# Qualora v0.2.0-alpha

Qualora v0.2.0-alpha adds an alpha API worker and OpenAPI checks to the self-hosted browser QA MVP.

This is still an early alpha. API testing is intentionally conservative and safe by default.

## What Works

- Run the stack locally with Docker Compose.
- Create projects with `frontend_url`, `api_base_url`, and/or `openapi_url`.
- Start runs that enqueue browser and API worker jobs.
- Run safe API baseline checks against `api_base_url`.
- Fetch and parse OpenAPI 3.x JSON/YAML documents.
- Check only safe OpenAPI methods by default: `GET`, `HEAD`, and `OPTIONS`.
- Record API observations, OpenAPI summaries, and API request evidence.
- Create findings for unreachable APIs, invalid OpenAPI documents, 5xx responses, unexpected status codes, obvious content type mismatches, and visible stack traces.
- Run deterministic browser and API smoke tests.

## Quick Start

```bash
docker compose up -d --build
make smoke
```

If local port `8080` is already in use:

```bash
QUALORA_API_PORT=18080 docker compose up -d --build
QUALORA_API_URL=http://localhost:18080 make smoke
```

## Safety Notes

Only run Qualora against systems you own or are explicitly authorized to test.

The API worker respects project `allowed_hosts`, blocks unsafe targets by default, and only tests `GET`, `HEAD`, and `OPTIONS` operations from OpenAPI documents. Destructive API testing is not enabled in this alpha.

## Not Included Yet

- Web UI.
- Authentication.
- Authenticated API testing.
- Login automation.
- Request body generation.
- Full OpenAPI schema validation.
- Schema fuzzing.
- Active security scanning.
- Helm/Kubernetes deployment.
