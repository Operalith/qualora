# Qualora v0.20.0-alpha

`v0.20.0-alpha` adds authenticated read-only API smoke testing and lightweight OpenAPI contract validation.

## Highlights

- Project-scoped API auth profiles with encrypted bearer token, API key, basic auth, and no-auth modes.
- Safe API auth profile CRUD and test endpoints.
- Authenticated API smoke runs for imported OpenAPI specs.
- Lightweight response contract checks for status, content type, JSON parseability, required fields, simple types, enum, nullable, and simple arrays.
- Optional unauthenticated status-only comparison for authenticated safe operations.
- API smoke JSON/HTML report updates with safe auth and contract metadata.
- Safe QA optional API smoke/contract integration when a parsed API spec is available.
- Web UI controls for API authentication profiles and authenticated API smoke runs.
- Demo API protected endpoints using the local-only `demo-api-token` fixture.

## Safety Model

Authenticated API smoke remains conservative:

- Only `GET`, `HEAD`, and `OPTIONS` operations are eligible.
- Mutating methods, required request bodies, unresolved parameters, sensitive paths/parameters, and destructive operations are skipped.
- API auth secrets are encrypted with `QUALORA_ENCRYPTION_KEY`.
- Raw bearer tokens, API keys, passwords, Authorization headers, request bodies, and response bodies are not stored or returned.
- AI analysis, CI output, and issue export previews use sanitized metadata only.

## Verification

Before tagging, run:

```bash
make test
make lint
docker compose config
docker compose down -v --remove-orphans
docker compose up -d --build
make smoke
curl -fsS http://localhost:8080/healthz
curl -fsS http://localhost:3000/healthz
git diff --check
docker compose down -v
```

The smoke flow covers first-run setup/login, browser smoke, authenticated browser smoke, discovery, quality checks, Safe Explorer, Safe QA, API auth profile creation/test, unauthenticated API smoke, authenticated API smoke, deterministic contract mismatch findings, AI analysis/planning, baselines, CI gates, CI run scripts, issue export dry-run, UI bundle text, and secret redaction.

## Known Limitations

- Authenticated API testing is alpha and limited to configured API auth profiles plus safe read-only OpenAPI operations.
- Contract validation is lightweight and not a full OpenAPI validator.
- No request-body validation, request generation, response-body storage, fuzzing, active scanning, payload attacks, or destructive testing is included.
- CI integration remains HTTP/script based rather than a full CLI.
- Only OpenAI-compatible chat completion providers are supported.
- No autonomous AI browser control is included.
