# Qualora v0.23.0-alpha

Qualora v0.23.0-alpha adds Qualora Demo Lab, a dedicated deterministic showcase target for validating the platform end to end without external services or real data.

## Highlights

- New `demo-lab-web` target with public pages, deterministic authentication, role-aware routes, safe GET forms, unsafe/mutating form fixtures, discovery actions, and passive quality issues.
- New `demo-lab-api` target with an OpenAPI 3.0 document, public and bearer-authenticated read-only endpoints, required-parameter skips, mutating-operation skips, an intentional contract mismatch, and deterministic server errors.
- New `demo-lab` Compose profile with manual host access and opt-in `baseline` or `regressed` modes.
- New `scripts/run-demo-lab.sh` one-command workflow and `make showcase-smoke` comprehensive validation target.
- Existing smoke assertions now support target-specific deterministic login selectors and success text while continuing to validate the original demo services.
- Fake LLM discovery planning recognizes stable search and product filter routes and continues to provide safe navigation/form plus unsafe policy-block fixtures.
- Dashboard showcase copy and guided setup defaults for Demo Lab.
- Full Demo Lab operator, safety, development, Compose, roadmap, changelog, and release documentation.

## Safety

Demo Lab is local-only test infrastructure. Its accounts and API tokens are fake; mutation routes return `405`; no real user, payment, transfer, or external service exists. Existing Qualora allowlist, same-origin, safe-method, deterministic form, and AI Browser Control policy gates remain authoritative. Reports and scripts must not expose passwords, bearer tokens, cookies, browser storage, auth headers, or session material.

## Known Limitations

- Demo Lab is a deterministic showcase target, not a production application or substitute for real-world validation.
- Intentional issues are safe and documented, and exact report counts can change when page/workflow limits change.
- Regression mode is deterministic and local-only; switching modes requires recreating the two Demo Lab services.
- The comprehensive showcase is longer than the core smoke because it exercises nearly every existing workflow.
- No destructive testing, active scanning, payload attacks, or fuzzing was added.

## Run It

```bash
scripts/run-demo-lab.sh
```

Then open `http://localhost:3000` for Qualora or `http://localhost:18085` for the target application. See `docs/demo-lab.md` for users, API tokens, expected findings, regression mode, and walkthrough guidance.
