# Qualora v0.24.0-alpha

Qualora v0.24.0-alpha focuses on a clearer public-alpha experience and observable browser testing.

## Highlights

- Simplified authenticated dashboard with a prominent Demo Lab showcase path.
- Project Cockpit with primary Safe QA, browser, AI control, discovery, and quality actions.
- Reusable near-live/replay Run Viewer for AI Browser Control and Safe Explorer.
- Step timelines with local screenshots, action metadata, AI suggestions, policy decisions, execution results, and blocked reasons.
- Post-action screenshots for executed AI Browser Control actions.
- Project-grouped report landing page and clearer report navigation.
- Demo Lab wizard choice between deterministic Fake LLM and an existing real provider.
- Optional `scripts/run-demo-lab-real-llm.sh` workflow for OpenAI-compatible providers.
- Updated deterministic smoke coverage for viewer copy, evidence metadata, screenshot downloads, and secret redaction.

## Safety

Browser control remains policy-gated and same-origin/allowlist constrained. AI proposes one typed action at a time and never controls Playwright directly. Credentials, cookies, browser storage, authorization headers, tokens, screenshots, full HTML, request bodies, and response bodies are not sent to AI.

The real LLM demo is opt-in, may incur provider cost, and is not used by smoke or CI.

## Known Limitations

- Run Viewer is replay/near-live polling, not video streaming.
- The unified viewer initially supports AI Browser Control and Safe Explorer; other browser workflows link to their existing reports.
- Real LLM mode supports OpenAI-compatible chat completion providers only.
- No arbitrary form submission, destructive testing, active scanning, payload attacks, or fuzzing is included.
