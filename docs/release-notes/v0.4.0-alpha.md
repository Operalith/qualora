# Qualora v0.4.0-alpha

Qualora v0.4.0-alpha focuses on making browser smoke testing real, deterministic, and easier to demonstrate.

## Highlights

- Browser-only smoke run endpoint: `POST /api/v1/projects/{project_id}/browser-smoke-runs`.
- Web UI button on project details: `Run browser smoke test`.
- Playwright browser observations now include target URL, final URL, status code, page title, body text length, timeout state, console errors, failed requests, and blocked requests.
- Screenshot evidence metadata now includes filename, object key, content type, size, storage backend, and created timestamp.
- Screenshot preview/download through `GET /api/v1/evidence/{evidence_id}`.
- Local `demo-web` smoke target so browser smoke does not depend on external internet access.
- `make smoke` validates JSON reports, HTML reports, browser screenshot evidence metadata, screenshot download, and the API/OpenAPI smoke target.

## Safety Notes

- Browser and API checks still enforce project `allowed_hosts`.
- Private/local targets remain blocked by default unless `allow_private_targets` is explicitly true.
- Evidence downloads are limited to evidence records already known to Qualora.
- The API and web UI still have no authentication; use them only in trusted local/self-hosted alpha environments.

## Known Limitations

- Browser smoke is basic alpha coverage, not a complete browser test suite.
- No login automation or authenticated browser flows.
- No Playwright trace export yet.
- No active security scanning.
- No authenticated API testing.
- Workers still write results directly to PostgreSQL.
