# Qualora v0.5.0-alpha

Qualora v0.5.0-alpha adds optional AI provider management and AI report analysis while keeping deterministic QA checks usable without any LLM.

## Highlights

- Configure OpenAI-compatible providers from the web UI or API.
- Use presets for OpenAI, OpenRouter, Ollama, and custom OpenAI-compatible gateways.
- Store provider API keys and extra headers encrypted at rest.
- Test provider connectivity with a minimal safe prompt.
- Run AI analysis for an existing completed run.
- View AI analysis in the web UI, JSON report, and HTML report.
- Run deterministic smoke verification with the local `fake-llm` service.

## Safety Notes

- AI is disabled until a provider is configured.
- Redaction is enabled by default.
- Screenshots, full HTML, cookies, credentials, authorization headers, and full network bodies are not sent to AI by default.
- The alpha web UI and API still have no authentication. Use only in trusted local/self-hosted environments.
- The Docker Compose `QUALORA_ENCRYPTION_KEY` default is for local demos only. Set a strong value before storing real provider credentials.

## Known Limitations

- OpenAI-compatible chat completions only.
- No native Anthropic, Gemini, or provider-specific SDK integration yet.
- AI analysis is synchronous in the control plane for this alpha.
- No autonomous AI browser control.
- No login automation or authenticated API testing.
- No active security scanning.

## Verification

Recommended release verification:

```bash
make test
make lint
docker compose config
docker compose up -d --build
make smoke
curl -s http://localhost:3000/healthz
curl -s http://localhost:8080/healthz
git diff --check
docker compose down -v
```
