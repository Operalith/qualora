# Qualora v0.6.0-alpha

Qualora v0.6.0-alpha adds AI-assisted test planning while preserving the alpha safety model: deterministic browser/API checks remain useful without AI, and generated plans are reviewable suggestions only.

## Highlights

- Generate AI-assisted test plans from sanitized project and run context.
- Store normalized plan JSON with assumptions, coverage goals, scenarios, steps, assertions, test data needs, instrumentation suggestions, and limitations.
- View, delete, and export test plans from the web UI.
- Link plans generated from a run into JSON and HTML reports.
- Validate planning with the deterministic local `fake-llm` smoke service.

## API

- `POST /api/v1/projects/{project_id}/ai-test-plans`
- `GET /api/v1/projects/{project_id}/test-plans`
- `GET /api/v1/test-plans/{test_plan_id}`
- `DELETE /api/v1/test-plans/{test_plan_id}`
- `GET /api/v1/test-plans/{test_plan_id}/export.json`

## Safety

- Qualora does not execute generated test plan steps.
- Redaction is enabled by default.
- Screenshots, full HTML, cookies, credentials, authorization headers, raw traces, and full network bodies are not sent to AI by default.
- The web UI and API still have no authentication and should only be used in trusted local/self-hosted alpha environments.

## Known Limitations

- AI test planning is alpha.
- Plans depend on the configured OpenAI-compatible provider and available sanitized evidence.
- No authenticated journey execution, login automation, autonomous AI browser control, active security scanning, destructive testing, schema fuzzing, or full test coverage claims.
