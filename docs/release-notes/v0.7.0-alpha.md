# Qualora v0.7.0-alpha

`v0.7.0-alpha` adds approved safe test plan execution.

Qualora still treats AI-generated plans as suggestions. This release can execute only a small deterministic browser DSL after a user explicitly previews and starts execution.

## Highlights

- Safe execution preview for generated test plans.
- Safe execution queue consumed by the browser worker.
- Persisted execution, scenario, and step status.
- Step-level skip reasons for unsupported or unsafe actions.
- JSON execution reports.
- Self-contained HTML execution reports.
- Web UI controls to preview, start, list, and inspect executions.
- Execution-linked findings and evidence.
- Deterministic smoke coverage for preview, execution, reports, and screenshot evidence download.

## Supported Safe DSL

- `goto`
- `assert_title_contains`
- `assert_url_contains`
- `assert_text_visible`
- `assert_element_visible`
- `assert_link_exists`
- `check_link_status`
- `capture_screenshot`
- `collect_browser_signals`
- `wait_for_load_state`
- `assert_no_console_errors`
- `assert_no_failed_requests`

## Safety Rules

- Generated plans are never executed automatically.
- Free-form model text is never executed as code or browser commands.
- Only `automation_candidate=true`, non-destructive, unauthenticated scenarios are eligible.
- Login, payment, submit, upload, mutation, admin, exploit, SQLi, XSS, SSRF, brute-force, and destructive steps are skipped.
- Navigation and link checks must stay on the project frontend origin and within `allowed_hosts`.
- `POST`, `PUT`, `PATCH`, and `DELETE` are not executed from test plans.

## Known Limitations

- Browser-only safe execution.
- No authentication or user management.
- No login automation.
- No form submission.
- No file upload.
- No active security scanning.
- No schema fuzzing.
- No autonomous AI browser control.
- No robust retry/recovery model for interrupted plan executions yet.
