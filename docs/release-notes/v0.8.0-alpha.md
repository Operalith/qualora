# Qualora v0.8.0-alpha

`v0.8.0-alpha` adds OpenAPI import and safe API smoke testing.

## Highlights

- Import OpenAPI 3.x specs from a project-scoped URL or inline JSON/YAML.
- Persist API spec metadata and discovered operations.
- Classify operations as safe or skipped with clear reasons.
- Run safe API smoke checks for imported specs.
- Execute only `GET`, `HEAD`, and `OPTIONS` operations by default.
- Skip mutating, authenticated, ambiguous, request-body, unresolved-parameter, and sensitive API operations.
- Record API check results with status, HTTP status, duration, content type, response size, errors, and skip reasons.
- Add API result tables to JSON and HTML run reports.
- Add web UI support for API spec import, spec detail, operation lists, safe API smoke runs, and API results.
- Add deterministic `demo-api` smoke service and OpenAPI spec.

## Safety

API testing remains conservative:

- No authentication support.
- No request bodies.
- No response body storage.
- No schema fuzzing.
- No destructive API testing.
- No active security scanning.
- No AI dependency for API execution.

AI analysis receives only sanitized API metadata and result summaries. It does not receive auth headers, cookies, tokens, request bodies, response bodies, API keys, or secret-like query values.

## Smoke Coverage

`make smoke` now validates:

- Browser smoke still works.
- AI provider test still works.
- AI analysis still works.
- AI test planning still works.
- Safe browser test plan execution still works.
- Demo OpenAPI import works.
- Operation discovery works.
- Unsafe operations are skipped.
- Safe API smoke execution works.
- `/broken` produces a deterministic 5xx finding.
- JSON and HTML API reports include API results.

## Known Limitations

- UI is alpha and intended only for trusted local/self-hosted environments.
- No authentication or authorization.
- No authenticated API testing.
- No login automation.
- No schema fuzzing.
- No destructive testing.
- No active security scanning.
- OpenAPI support is limited to OpenAPI 3.x.
- The legacy API worker still exists for project-level API jobs; the imported-spec API smoke path runs in the control plane for this alpha.
