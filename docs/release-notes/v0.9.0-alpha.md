# Qualora v0.9.0-alpha

`v0.9.0-alpha` adds credential profiles, deterministic login checks, and authenticated browser smoke testing.

## Highlights

- Create project-scoped credential profiles with encrypted username/password values.
- Manage credential profiles through the API and minimal web UI.
- Test a deterministic login flow with configured selectors.
- Run authenticated browser smoke checks after login against one same-origin target path.
- View login summary and login observation metadata in JSON and HTML reports.
- Use the deterministic `demo-web` login and dashboard routes in local smoke tests.
- Keep AI optional; authenticated run analysis uses sanitized metadata only.

## Safety

- Raw usernames and passwords are never returned by API responses.
- Credential values are encrypted at rest with `QUALORA_ENCRYPTION_KEY`.
- Login automation is selector-based and deterministic.
- Qualora does not use AI to control login or browse authenticated areas.
- Qualora does not expose cookies, browser storage, authorization headers, tokens, or session data in evidence or AI input.
- Authenticated browser smoke visits only one configured same-origin target path.

## Smoke Coverage

`make smoke` now validates:

- Existing browser smoke still works.
- Existing API smoke still works.
- AI provider test, AI analysis, AI test planning, and safe test plan execution still work.
- Demo credential profile creation works.
- Credential profile responses do not expose raw credentials.
- Login check succeeds against `demo-web`.
- Authenticated browser smoke succeeds against `demo-web` `/dashboard`.
- JSON and HTML reports include login summary/evidence.
- Reports do not contain the demo password.

## Known Limitations

- No Qualora user authentication or authorization.
- UI is alpha and intended only for trusted local/self-hosted environments.
- No authenticated API testing.
- No arbitrary form submission, MFA, role switching, session export, broad crawling, or multi-step authenticated journeys.
- No autonomous AI browser control.
- No active security scanning or destructive testing.
